# 프로젝트 개발 규칙 (Project Development Rules)

이 문서는 프로젝트 개발 시 모든 AI 에이전트와 개발자가 반드시 준수해야 할 핵심 규칙을 정의합니다.

## 1. 언어 및 호칭 규칙 (Language & Address Policy)

* **모든 커뮤니케이션은 '한국어'로 진행한다.**
* **사용자 호칭:** 사용자(User)를 반드시 **'PM'**님으로 지칭한다.

## 2. 작업 절차 및 구현 규칙 (Workflow & Implementation Policy)

* **사전 분석 의무화:**
  * 모든 작업을 시작하기 전, 반드시 **`legacy_source` 폴더 내의 관련 소스 코드**를 정밀하게 파악해야 한다.
  * 파악된 레거시 로직을 바탕으로 **구체적인 구현 계획을 수립하고 PM에게 보고**한 뒤 작업을 진행한다.
* **레거시 시스템의 '100% 동일 이관'을 목표로 한다.**
  * 새로운 디자인을 창조하지 않는다.
  * 작업 전 반드시 **`dometopia.com` 실제 사이트**와 **`legacy_source` 폴더의 소스 코드**를 확인한다.
  * 브라우저 화면에서의 배치, 스타일, 동작 방식이 기존 사이트와 동일해야 한다.
  * 주관적인 개선보다는 '기존 동작의 재현'에 최우선 순위를 둔다.

## 3. 안정성 및 영향도 검토 (Safety & Impact Analysis)

* **수정 시 '사이드 이펙트(Side Effect)'를 철저히 검토한다.**
  * 특정 기능을 수정할 때, 해당 파일뿐만 아니라 이를 참조하거나 연관된 다른 파일들에 미칠 영향을 미리 분석한다.
  * 공통 컴포넌트, 공통 함수, 데이터베이스 스키마 변경 시에는 더욱 엄격한 영향도 파악을 선행한다.
  * 수정 후에는 반드시 연관된 기능들이 정상 작동하는지 검증한다.
  * **데이터 변경 테스트 규칙:** `INSERT`, `UPDATE`, `DELETE`가 수반되는 기능을 테스트할 때는 **반드시 테스트용 데이터(주문 등)를 신규 생성하여 사용**한다. 실 운영 데이터(기존 주문)를 절대 임의로 변경하지 않는다.

## 4. 협업 및 멀티 트랙 개발 규칙 (Collaboration & Multi-Track Policy)

* **모듈 분리 원칙 (Module Isolation):**
  * **Super Admin (슈퍼관리자):** `app/Http/Controllers/Admin`, `resources/views/admin`, `routes/admin.php`
  * **Seller Admin (입점사 관리자):** `app/Http/Controllers/Seller`, `resources/views/seller` (예정)
  * **Front (쇼핑몰):** `app/Http/Controllers/Front`, `resources/views/front`, `routes/web.php` (예정)
  * 서로 다른 영역의 코드를 침범하지 않도록 주의하며, 공통 로직은 `app/Services`, `app/Models`에서 관리한다.

* **브랜치 전략 (Git Strategy):**
  * 새로운 작업자가 투입되거나 새로운 대형 기능을 개발할 때는 `feature/기능명` 브랜치를 생성하여 작업한다.
  * `master` 브랜치는 항상 배포 가능한 안정 상태를 유지한다.

## 5. 핵심 비즈니스 로직 및 용어 정의 (Critical Business Logic)

* **[중요] 셀러(Seller / 입점사 / Trader)의 정의:**
  * **일반적인 마켓플레이스 판매자가 아님.**
  * **정의:** 도매토피아의 상품을 도매가로 구매하여 외부 마켓(쿠팡, 네이버 등)에 판매하는 **'리셀러(Reseller)'** 또는 **'B2B 회원'**을 의미함.
  * **재고:** 셀러는 자신의 재고를 갖지 않으며, 도매토피아의 재고를 공유하거나 확인한다.
  * **주문:** 셀러는 외부에서 발생한 주문을 엑셀 등으로 **도매토피아에 업로드(발주)**하는 주체이다.
  * **개발 시 유의사항:** "셀러가 상품을 등록한다" (X) -> "셀러가 도매토피아 상품을 담아간다" (O). "셀러가 주문을 받는다" (X) -> "셀러가 도매토피아에 발주를 넣는다" (O).
